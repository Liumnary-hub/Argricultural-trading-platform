# 高并农业交易电子系统
基于 Spring Boot 开发的电商订单与库存管理核心系统，聚焦**高并发场景下的库存安全**、**数据一致性**和**接口性能优化**，通过 MySQL 悲观锁、Redis 缓存、Redis 分布式锁三重保障，彻底解决商品超卖、并发数据错乱、数据库压力过大等问题。

## 项目背景
针对电商下单、库存扣减等高频并发业务场景，传统无锁设计会引发库存超卖、数据不一致等问题；本项目结合**数据库悲观锁**保证数据库层面原子操作，**Redis 分布式锁**保证分布式集群环境下的并发安全，**Redis 缓存**降低数据库查询压力，实现安全、高效、稳定的订单创建与库存管理能力。

## 技术栈
- 核心框架：Spring Boot
- 数据持久层：MyBatis-Plus、MySQL
- 事务管理：Spring 声明式事务 `@Transactional`
- 缓存中间件：Redis
- 分布式锁：Redis 分布式锁
- 工具库：Lombok、SLF4J 日志

## 核心功能
1. **订单管理**：创建订单、订单金额自动计算、订单状态初始化
2. **库存管控**：库存校验、原子化扣减库存、防止超卖
3. **缓存优化**：商品信息 Redis 缓存，大幅提升查询效率，减轻数据库压力
4. **并发安全**：Redis 分布式锁 + MySQL 悲观锁双重锁机制，保证高并发下数据安全
5. **事务一致性**：订单创建和库存扣减绑定事务，要么全部成功，要么全部回滚
6. **异常处理**：全局异常捕获、业务错误提示、日志记录

## 核心技术实现
### 1. MySQL 悲观锁（行锁）
- 使用 `SELECT ... FOR UPDATE` 对商品库存行加排他锁
- 同一时间仅允许一个线程操作同一商品库存
- 从数据库底层保证库存扣减的原子性，**彻底防止超卖**

### 2. Redis 缓存
- 对热点商品数据进行缓存，查询时优先读取 Redis
- 减少数据库查询频次，提升接口响应速度
- 缓解高并发下数据库的查询压力

### 3. Redis 分布式锁
- 解决分布式/集群部署环境下，多服务实例的并发竞争问题
- 防止重复下单、超卖等分布式场景下的数据异常
- 配合数据库锁，形成**分布式锁+数据库锁**的双层安全保障

### 4. Spring 事务管理
- 通过 `@Transactional` 实现订单与库存的事务绑定
- 订单创建失败、库存不足、系统异常时自动回滚
- 保证业务数据的强一致性

## 项目特点
1. **高安全性**：双重锁机制 + 事务管理，杜绝超卖、数据错乱
2. **高性能**：Redis 缓存优化查询性能，适配高并发下单场景
3. **分布式适配**：支持集群部署，通过分布式锁保证多实例安全
4. **健壮性**：完善的业务校验、异常处理和日志记录
5. **易扩展**：代码结构清晰，可快速对接支付、物流等模块

## 数据库核心表
1. **商品表(product)**：存储商品ID、名称、库存、价格等信息
2. **订单表(order)**：存储订单ID、商品ID、购买数量、订单总价、状态等信息

## 快速启动
### 1. 环境准备
- JDK 8+
- MySQL 5.7+
- Redis 5.0+
- Maven 3.6+

### 2. 配置修改
1. 创建数据库，执行商品表、订单表建表 SQL
2. 修改 `application.yml` 配置：
   - MySQL 数据库连接地址、账号、密码
   - Redis 连接地址、密码

### 3. 项目启动
1. Maven 依赖安装
2. 启动 Spring Boot 主类
3. 调用订单创建接口进行测试

## 核心接口
### 创建订单接口
- 地址：`/order/create`
- 方式：POST
- 功能：校验商品/库存 → 加锁 → 创建订单 → 扣减库存 → 释放锁
- 返回：订单信息 / 错误提示

## 注意事项
1. 确保 Redis 服务正常运行，否则分布式锁和缓存功能失效
2. 悲观锁仅作用于单行数据，不会影响其他商品的并发操作
3. 分布式锁需合理设置过期时间，避免服务宕机导致锁无法释放
