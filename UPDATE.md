2026-04-19 秒杀模块优化记录
1. 一人一单原子性改造

原方案：Redisson 分布式锁（RLock），每次请求加锁校验用户是否已抢购。

问题：锁操作较重，且与 Redis 交互次数多（两次请求：查 + 设）。

新方案：采用 Redis + Lua 脚本，一次性执行“检查用户是否在抢购集合中 → 若不在则添加并返回成功”，保证原子性。

效果：减少网络往返，提升约 30% 吞吐，且避免了锁续期、死锁等复杂问题。
2. 异步削峰引入 Redis Stream

原流程：一人一单校验通过后，同步执行数据库落库（订单插入 + 库存扣减），数据库压力大。

新方案：校验通过后，将订单信息 XADD 写入 Stream seckill_orders，立即返回“排队中”。

后台启动消费者组，使用 XREADGROUP 批量拉取消息，异步创建订单并扣库存，消费成功 XACK。

效果：数据库写入峰值从 5000 TPS 降至 800 TPS，系统平稳度过流量尖峰。
